Intro：操作对象：寄存器、内存和常数

指令（instruction / command）对操作数（operands）进行操作。在 sub a, b, c 中，变量 a、b 和 c 都是 operand。但是计算机在 1 和 0 上运行，而不是变量名称。指令需要一个物理位置来检索二进制数据。operands可以存储在寄存器（register）或内存（memory）中，或者它们可能是存储在指令本身中的常量。计算机使用不同的位置来保存operands，以优化速度和数据容量。访问存储为常量或 register 中的 operands 很快，但它们只保存少量数据。其他数据必须从内存访问，内存很大但速度很慢。RISC-V被称为 32 位体系结构，因为它在 32 位数据上运行。

1. 基本命令用法

1.1 寄存器

类似c语言中的变量，存储数据、地址等信息。

1.2 基本运算

1.2.1 寄存器运算

语句格式为 <cmd> <reg0>, <reg1>, <reg2> ，功能为将两个寄存器 <reg1>, <reg2> 的值进行一定规则的运算后赋值给 <reg0> 。

命令 功能
add （有符号）加法
sub （有符号）减法
and 按位与
or 按位或
xor 按位异或
sll (shift left logical) 左移（低位空缺补0）
srl (shift right logical) 右移（高位空缺补0）
sra (shift right arithmetic) 算数右移（高位空缺补符号位）
mul 乘法，取低32位
mulh (multiply high signed signed) 有符号数乘法，取高32位
mulhsu (multiply high signed unsigned) 有符号数乘无符号数，取高32位
mulhu (multiply high unsigned unsigned) 无符号数乘法，取高32位

*由模意义下的乘法规则，低位乘法同时适用于有、无符号数乘法

寄存器运算的对象（operands）必须是寄存器。

1.2.2 即时操作

除了寄存器操作外，RISC-V指令还可以使用常量（或称即时操作数 Immediates）。这些常量称为即时常量，因为它们的值可立即从指令中获得，并且不会需要寄存器或内存访问。在汇编中， Immediates 可以用十进制、十六进制或二进制。RISC-V 汇编语言中的十六进制常量以 0x 、二进制常量以 0b 开头，就像它们在 C 中一样。addi 指令中， Immediates 是 12 位二进制的补码编号，因此它们被符号扩展为 32 位。

addi 即时加法，addi s0, zero, 53 相当于高级语言的 s0=53

注意符号扩展：Immediates 是 12 位二进制的补码， 故会在执行加法前进行符号扩展.

addi s0, s0, 0x987 相当于高级语言的 s0=s0+0xFFFFF987

要创建更大的常量，请使用 lui 后跟 addi 。lui 指令加载 20 位 Immediates ，将其放在 most significant bits 上并且后面补 0。

注意：创建大型 Immediates 时，如果 addi 中的 12 位 Immediates 为负数（即位 11 为 1），lui 中的 Immediates 上半部分必须+1。

lui (load upper immediate)，lui s0, 0xABCDE 相当于高级语言的 s0=0xABCDE000

上例中，为创建 int s2=0xFEEDA987 ，将 upper immediate +1 以抵消低位的符号位扩展（0xFEEDB000+0xFFFFF987==0xFEEDA987）。

命令及语法 功能
lui <reg>, <20-bit unsigned> 赋值（高位）
addi <reg>, <12-bit signed> 加法（符号扩展）
andi <reg>, <12-bit signed> 按位与（符号扩展）
ori <reg>, <12-bit signed> 按位或（符号扩展）
xori <reg>, <12-bit signed> 按位异或（符号扩展）
slli <reg>, <5-bit unsigned> 左移
srli <reg>, <5-bit unsigned> 右移
srai <reg>, <5-bit unsigned> 算数右移

1.3 内存访问

在RISC-V架构中，指令仅在寄存器上运行，因此存储在内存中的数据必须在处理之前移动到寄存器上。

RISC-V的内存按字节寻址，即每个 byte 有一个特定的地址。一个32-bit的字（word）由4个 byte 组成，故字的地址是4的倍数。通常地址和其上的内容写成16进制的格式。如：字 0xF2F1AC07 储存在地址 4 上。

lw (load word) 指令将数据从内存读入寄存器中，格式为 lw <reg>, <offset>(<base address>) 。如 lw s7, 8(zero) 将地址为8的字读入寄存器 s7 中。

sw (store word) 指令将数据从寄存器写入内存中，格式为 sw <reg>, <offset>(<base address>) 。

上面代码在高级语言可以表示为 mem[5]=mem[2] （mem为32-bit数据, s0为mem base address）

命令及语法 功能
lw <reg>, <offset>(<base address>) 按字读取内存
sw <reg>, <offset>(<base address>) 按字写入内存
lb <reg>, <offset>(<base address>) 按字节读取内存（符号位扩展）
lbu <reg>, <offset>(<base address>) 按字节读取内存（高位补0）
sb <reg>, <offset>(<base address>) 按字写入内存（取寄存器低位）

1.4 流程控制

1.4.1 条件分支

语句格式为 <cmd> <reg1>, <reg2>, <label> ，可在两个寄存器变量 <reg1>, <reg2> 满足 <cmd> 规定的特定条件时跳转至 <label> 处。

命令 分支条件
beq (branch if equal) 相等
bne (branch if not equal) 不相等
blt (branch if less than) 小于
bge (branch if greater than or equal to) 大于等于
bltu and bgeu treat the operands as unsigned. 小于/大于等于 （将operands视为无符号类型）

1.4.2 跳跃

j ，语句格式为 j <label> ，可跳转至 <label> 处。

1.5 函数调用（基本用法）

jal <callee> (jump and link), 将返回地址填入 ra , jr <return address reg> (jump register) 可根据返回地址使程序流程返回main函数中。

以上仅为函数的简单调用。函数的嵌套、递归调用由于比较复杂，将单独开一篇博客详细说明。
